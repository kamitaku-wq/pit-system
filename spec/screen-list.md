# A. 共通方針

- **モバイル対応**（タブレット / スマホ）必須 — 現場・工場・回送業者がスマホで使用
- **印刷レイアウト** — 整備伝票 / 回送依頼書 / 店間移動指示書
- **検索・フィルタ** — 一覧画面はすべて検索・絞り込み・並び替え可
- **CSV エクスポート** — 主要一覧画面に対応
- **アクセシビリティ** — WCAG 2.1 AA を目標（顧客予約画面）
- **会社別表示** — ヘッダに会社名 + 所属店舗バッジ表示
- **楽観排他のコンフリクト UI**（v2.1）— UPDATE が `OptimisticLockError` を返した場合、モーダルで「他のスタッフが先に更新しました。最新内容を確認して再操作してください」と表示し、再読込ボタンを提示
- **PII 表示制御**（v2.1）— 監査ログ閲覧・通知履歴での電話 / メール / VIN はマスク表示（権限により表示可、要監査ログ記録）

# 4. 業者用マイページ（vendor_users 認証）

## 4.3 対応可否回答

- 対応可 / 対応不可
- 対応不可時の理由入力
- 備考
- **先着受注の競合通知（v2.1）**：複数業者打診時、他業者が先に accept した場合は「他業者が既に受注済み」と表示し、自業者の招待は revoked

## 4.3.1 招待トークン経由の受諾画面（v2.1、v2.2 で未登録業者対応強化）

スポット業者・未登録業者向け。招待トークン URL 経由でアクセスし、ログインなしで案件詳細を確認 → 受諾可能。

### 4.3.1a 招待画面（未認証）

- 招待トークン検証（`invitation_token_hash`）
- 期限切れチェック
- 案件詳細表示（車種・移動経路・希望日時）
- 「対応可」「対応不可」ボタン
- **承諾時の同意チェックボックス**（v2.2）：「以下の内容で回送業務を引き受けます」（責任分界の証跡）

### 4.3.1b 受諾フロー（未登録業者の場合、v2.2）

1. 「対応可」+ 同意チェック → 業者情報入力フォーム（業者名・連絡先・担当者名、`invitee_*` 列に保存済みなら自動入力）
2. Supabase Auth 招待メール送信（パスワード設定リンク付き）
3. パスワード設定完了 → `vendor_users` 作成 + `vendors` 新規作成（未登録の場合）
4. `transport_order_invitations.bound_vendor_id` / `bound_vendor_user_id` を紐付け
5. `accept_invitation_and_revoke_others()` DB 関数を呼ぶ
6. `transport_orders.vendor_id` セット、他招待は revoke
7. 業者ポータルへリダイレクト

### 4.3.1c 受諾フロー（登録済業者の場合）

1. 「対応可」+ 同意チェック → 業者ログイン画面へ
2. ログイン後、`accept_invitation_and_revoke_others()` 呼び出し
3. 業者ポータルへリダイレクト

### 4.3.1d 競合エラー時の表示（v2.2）

- 「他業者が既に受注済みです」
- 「招待は失効しました」
- 連絡先（販売会社の担当店舗）表示

## 4.4 進捗更新

- 引取予定日時入力
- 搬入予定日時入力
- 返却予定日時入力
- 引取完了報告
- 搬入完了報告
- 返却完了報告
- キャンセル

各遷移は `status_transitions` で許可された遷移のみ可。
