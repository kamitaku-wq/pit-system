## 起動方法

### Option A: npx serve（Node.js 環境、ポート 5175）

```bash
cd design-v3
npm run dev
```

→ `http://localhost:5175` で開く

### Option B: Python 標準モジュール

```bash
cd design-v3
npm run py
# または
python -m http.server 5175 --directory project
```

### Option C: VS Code Live Server 拡張

`design-v3/project/index.html` を右クリック → "Open with Live Server"

## 既知の参照ドキュメント残存

`project/uploads/claude-design-handoff.md`（=入力資料コピー）には v2 時点の表現が一部残存：
- 「SaaS：整備ピット予約 + 店間整備 + 業者通知」（line 28）
- 「モダン・軽快な SaaS 調」（line 33）
- 「M2M で対応エリア / 店舗 / 曜日を編集。`is_shared` チェックボックス」（line 981）

これらは **UI に表示されない参照ドキュメント**であり、Claude Design への入力履歴として残されているもの。実画面ファイルからは完全駆逐済みのため、営業資料への影響なし。

## v3 で確定した最終方針

1. システムは「貴社専用」として見せる
2. 他社 / マルチテナント / SaaS / 共有という存在を顧客に開示しない
3. 業者マスター `is_shared` フラグは DB 内部のみ、UI 露出ゼロ
4. セキュリティ訴求は ISO27001 / 法令準拠の語彙で
5. 内部設計文書（requirements.md / data-model.md）は維持
