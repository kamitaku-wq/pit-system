# §A. ブランドガイドライン

## A.1 プロダクト概要

| 項目 | 内容 |
|---|---|
| サービス名 | **段取りくん** |
| 英字表記 | **Dandori-kun**（必要時） |
| タグライン | 面倒な段取りを、段取りくんが全部やってくれる。 |
| 領域 | 中古車販売会社向け SaaS：整備ピット予約 + 店間整備 + 業者通知 |
| 対象ユーザー | 多店舗展開する中古車販売会社の経営層・現場スタッフ・回送業者・顧客 |

## A.2 トーン

- **モダン・軽快な SaaS 調**（Notion / Linear / Vercel 系の現代的フラットデザイン）
- **白基調 + 余白多め**
- **名前は親しみやすく、UI は信頼感重視**のバランス
- マスコット・キャラクターは使わない（or 最小限のロゴマーク程度）

## A.5 スペーシング

8px グリッドベース。`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`px を基本とする。

- セクション間: 48px
- カード内 padding: 24px
- フォーム要素間: 16px
- ボタン padding: 12px 24px

## A.6 角丸・影

- Border radius: `4px`（小要素）/ `8px`（カード）/ `12px`（モーダル）
- Shadow: `0 1px 2px rgba(15, 23, 42, 0.06)`（カード）/ `0 10px 25px rgba(15, 23, 42, 0.10)`（モーダル）

## A.7 アイコン

- **Lucide Icons** を採用（線画、22px / 16px、stroke 1.5px）
- ステータスバッジ用の専用アイコン:
  - 対応可 → CheckCircle2
  - 対応不可 → XCircle
  - 業者未確認 → Clock
  - 通知失敗 → AlertTriangle
  - 移動中 → Truck
  - 完了 → Flag

## B.4 ボタン

| 種類 | 用途 | スタイル |
|---|---|---|
| Primary | 確定、保存、送信 | 濃紺背景 + 白文字 |
| Secondary | キャンセル、戻る | 白背景 + 濃紺枠 + 濃紺文字 |
| Tertiary | リンク的アクション | テキストのみ（濃紺） |
| Danger | 削除、キャンセル予約 | 赤背景 + 白文字 |
| Ghost | ツールバー内 | 透明 + hover で薄グレー |

サイズ: lg (48px) / md (40px) / sm (32px)

## B.5 フォーム要素

- Input: 40px 高さ、border radius 4px、focus 時に濃紺リング 3px
- Select: Combobox 風、検索可能（react-select / cmdk）
- Date / Time picker: ポップオーバー型
- Textarea: 縦リサイズ可
- Checkbox / Radio: カスタムスタイル、濃紺チェック

## B.6 テーブル

```
┌──────────────────────────────────────────────────────┐
│ □ │ 列1     │ 列2 ▾  │ 列3        │ 列4     │ アクション │
├──┼─────────┼────────┼────────────┼─────────┼──────────┤
│ □ │ ...     │ ...    │ [バッジ]   │ ...     │ [⋮]      │
└──────────────────────────────────────────────────────┘
```

- ヘッダ：薄グレー背景、太字、ソート矢印
- 行：ホバーで `#F8FAFC` 背景、選択中はチェックボックス + 薄濃紺
- 行 padding: 12px 16px
- アクションは右端の縦三点ドット（メニュー展開）

## B.7 ステータスバッジ

色 + アイコン + テキストの組み合わせ:

```
[●対応可]     ← 緑背景 + チェックアイコン
[●未確認]    ← 黄背景 + 時計アイコン
[●対応不可]  ← 赤背景 + ×アイコン
[●移動中]    ← 青背景 + トラックアイコン
[●完了]      ← グレー背景 + フラグアイコン
```

## B.8 モーダル / ダイアログ

- 中央配置、最大幅 600px（大は 800px）
- オーバーレイ: `rgba(15, 23, 42, 0.5)`
- 角丸 12px、shadow lg
- 右上に閉じる×ボタン
- footer に Secondary（左）+ Primary（右）の 2 ボタン

## B.9 通知トースト

- 右下から出現、4 秒で自動消滅
- 成功（緑）/ 警告（黄）/ エラー（赤）/ 情報（青）

# §J. ハンドオフバンドル要求

Claude Design から **以下の形式** でアウトプットをもらいたい：

1. **コンポーネントライブラリ**（shadcn/ui ベース、React + TypeScript + Tailwind）
2. **デザイントークン**（`tokens.json` 形式、カラー / フォント / スペーシング / 影）
3. **画面 HTML プロトタイプ**（各画面 1 ファイル、状態違いも含む）
4. **コピーライティング**（マイクロコピー、エラーメッセージ、空状態メッセージ）
5. **画像 / イラスト**（空状態用、エラー用、オンボーディング用、各画面 1-2 点）
6. **Figma ファイル**（あれば）

# §L. NG / 注意事項

- マスコットキャラクターは作らない（or 最小限のロゴマーク）
- アイコンや色だけで意味を伝えない（必ずテキスト併用）
- 個人情報（電話 / メール / VIN）はモックデータでもプレースホルダ風（`090-xxxx-xxxx`、`u***@example.com`）
- 業者ポータルは「外部利用者」前提のシンプル UX（管理画面と明確に差別化）
- 過度なアニメーション / マイクロインタラクションは避ける（BtoB 業務、軽快感は余白とリズムで出す）
