1.はじめに

段取りくんは、中古車販売会社の整備業務を「ピット予約・店間移動・業者連携・顧客予約」のひとつのクラウドにまとめる業務システムです。本資料では、何ができて、どこをどう操作すれば、どんな動きになるかを画面とともに説明します。

1.1 段取りくんとは

整備ピットを持つ店舗と持たない店舗が混在する販売会社向けに、整備の段取りを一元管理するクラウドシステムです。「電話・FAX・付箋・Excel」で属人化していた段取り業務をシステム化し、業者連携と顧客予約までを 1 つの画面で扱えるようにします。

1.2 主要機能

機能 01

ピット予約共有

全店舗のピット空き状況をカレンダー 1 枚で。レーン種別・作業内容ごとの稼働も可視化。

機能 02

店間整備予約

ピットがない店舗から、ピットがある店舗へ即予約。移動パターン 4 種（片道 / 往復 / 引取のみ / 三点）に対応。

機能 03

業者通知の自動化

店間移動の業者依頼を自動メール + マイページで通知。対応不可時は次候補へ再打診。

機能 04

顧客来店予約

お客様は 24 時間 Web から予約可能。電話受付の代行入力にも対応。

2.業務シナリオ別ガイド

日常業務でよく発生する 8 つのシナリオについて、画面とともにステップを追って説明します。各シナリオは「想定シーン → 関わる人 → 全体フロー → ステップ詳細（スクショ + 注釈 + 動作結果）」の統一テンプレートです。

S01 顧客から電話で予約を受ける

平日午前 10:30、お客様から店舗に電話。「来週土曜の午後にオイル交換をお願いしたい」との問い合わせ。受付スタッフは電話を切らずに、段取りくんで予約を入力します。

店舗受付スタッフ
システムを操作して予約を入力
顧客
電話相手、システム操作なし

全体フロー

  1. サイドナビ「顧客予約」を開く
  2. 一覧画面上部の「+ 電話受付で追加」を選ぶ
  3. 顧客情報・車両情報を聞き取りながら入力
  4. 空き日時カレンダーから土曜午後を選択
  5. 予約確定 → 顧客に予約番号を伝える

  • 4「14:00 オイル交換（約 30 分）」スロットを選択
動作結果：仮予約状態で時間枠が確保される（既定 30 分間の仮確保。設定で変更可）。同じ時間を別スタッフが選ぼうとすると競合検出される。

S02 顧客が Web から自分で予約する

夜 22 時、顧客がスマホで段取りくんの顧客予約 URL を開きます。希望する作業を選び、空き日時を確認して予約。本人確認の認証コードを入力すると予約完了です。

顧客

システム
空き枠検索・認証コード送信・確認メール送信を自動実行

全体フロー（6 ステップ）

  1. 顧客が URL を開き、店舗を選ぶ
  2. 希望する作業メニューを選ぶ
  3. 空き日時カレンダーから選択
  4. 顧客情報・車両情報を入力
  5. 認証コードがメールで届く → 6 桁を入力
  6. 予約完了。確認・変更・キャンセル URL がメールで届く

  • 3各カード：カテゴリ / メニュー名 / 所要時間（例「約 30 分」）/ 料金目安（例「¥4,800〜」）

ステップ 3

  • 1ステップ進行「3 / 6」、月切替（前月 ← → 翌月）
  • 2曜日 × 日付の横並びピッカー、各日付下に空き状況のインジケータドット

動作結果：緑のスロットをタップすると選択中表示に。下部のサマリで内容を確認して次ステップへ。

  • 1ステップ進行「4 / 6」、選択中の予約内容が上部に固定表示

  • 3車両情報：車種・ナンバー
  • 4「利用規約」「プライバシーポリシー」への同意チェック
  • 5下部「認証コード送信」ボタン
動作結果：入力したメールアドレス宛に 6 桁の認証コードがメール送信される。
ステップ 5

認証コードを入力

  • 1ステップ進行「5 / 6」、「認証コードを入力」見出し
  • 2送信先メールアドレス（マスク表示 t****@example.com）の確認
  • 36 桁の正方形ボックス、タップで OS のキーボード起動、OTP 自動入力にも対応
  • 4「再送信（残り 4 回 / 本日）」リンク、レート制限を可視化
  • 5次回送信までのカウントダウン表示
動作結果：コードが正しければ予約確定 → ステップ 6 へ。レート制限超過時は §E.5 認証コード再送制限画面へ。
ステップ 6

  • 1大きなチェックマーク + 「ご予約完了」
  • 2予約番号（例 R-2026-0042）・日時・店舗・作業・料金目安がカード表示、ステータス「確定」
  • 3「店頭でこの QR をご提示ください」案内 + QR コード画像
  • 4「カレンダーに追加」ボタン
  • 5下部に「予約内容を変更 / キャンセル」リンク（署名付き URL で個別認証）
動作結果：顧客にはメールで「予約確認」「変更」「キャンセル」用の署名付き URL が届き、ID/Pass なしで操作可能。

S03 自店にピット空きがない → 店間整備依頼

渋谷店スタッフがお客様の車検整備を受けましたが、当日のピットがすべて埋まっています。横浜整備工場に空きがあったので、店間整備として依頼します。

店舗スタッフ（渋谷店）
店間整備依頼を作成、業者を選んで通知まで実行
顧客
既に来店済み or 電話受付済み（このシナリオでは操作なし）

全体フロー（7 ステップ）

  1. サイドナビ「店間整備依頼」を開く
  2. Step 1: 整備伝票番号と車両情報を入力
  3. Step 2: 作業メニュー選択 → 空き日時・レーン検索
  4. Step 3: 店間移動の有無と移動パターンを選ぶ
  5. Step 3 続き: 業者を選択（単一指名 / 一斉打診 / スポット招待）
  6. 「予約確定 + 業者通知」ボタンで一気に確定
  7. 業者状況を C.5 業者通知・回送管理で確認

S04 店間移動の業者通知が自動で飛ぶ

5.用語集

段取りくんで使われる主な業務用語とシステム用語の定義です。

整備伝票
1 件の整備案件を識別する伝票。車両 / 顧客 / 作業内容 / 担当者 / 入庫予定 / 実績 / 金額が紐付く中核データ。番号例：SV-2026-0089
ピット
整備作業を行う作業スペース。1 店舗に複数あり、種別（メンテナンス / 重整備 / 汎用）と稼働時間が設定可能
レーン
ピット内の予約単位。同時に作業できる枠数（concurrent_capacity）を持つ
店間整備
自店にピット空きがない時、他店のピットで整備を行うこと。本システムの中核機能の 1 つ
店間移動
店間整備に伴う車両の物理的な移動。回送業者または陸送業者が担う
回送 / 陸送
回送 = 業者が自走して車両を運ぶ / 陸送 = キャリアカー等での運搬。本システムでは両者をまとめて扱う
仮予約
確定前の予約状態。設定で有効期限（例 30 分）を超えると自動失効
確定
予約確定。この時点で関連業者に通知が飛ぶ（仮予約時点では飛ばない）
作業中 / 完了
整備士が現場で更新するステータス。「次に何をやるか」のカンバンに連動
移動中
業者が車両を引取・搬入中の状態
移動パターン
店間移動の 4 種類：片道（引取→搬入）/ 往復（引取→搬入→返却）/ 引取のみ（廃車送り等）/ 三点（引取・搬入・返却が異なる店舗）
確定モード（auto / manual）
業者が「対応可」を返した時の店舗側挙動。auto = 自動確定 / manual = 店舗承認後に確定
走行可否
車両が自走可能か。不可の場合 tow_required（レッカー必要）が自動セットされ、業者通知にも明記
業者ポータル
回送業者向けの専用画面。社内ユーザーとは認証基盤上完全に分離
招待 URL
未登録の業者を案件単位で招待する仕組み。受諾と同時に業者アカウントが発行される
承諾証跡
業者「対応可」回答時に記録される送信時刻・IP・User-Agent・同意フラグ。トラブル時の責任分界証跡として機能
認証コード
顧客の予約完了時にメールで送信される 6 桁コード。本人確認用。有効期限 5 分
署名トークン URL
予約完了メールに記載される個別 URL。顧客は ID/Pass なしで自分の予約を確認・変更・キャンセル可能
本人確認 / KYC
顧客の身元確認。MVP では email 認証コード方式、将来 SMS や eKYC への拡張に対応
通知ルール
「いつ / 誰に / 何で送るか」の組み合わせ設定。設定画面から追加・編集可能
通知配送（outbox）
通知をデータベースに記録 → 配送ワーカーが順次送信する設計。失敗時はリトライ + 失敗一覧で可視化
同時更新の競合
複数スタッフが同じデータを同時編集した時の検出。後から保存しようとした人にはモーダルが出て、相手の変更内容を確認できる
操作記録
誰がいつ何を変更したかの全件記録。本部管理者のみ閲覧可。改ざん不可
役割 / 権限
本部管理者 / 店長 / 工場長 / 店舗スタッフ / 整備士 / 業者 / 顧客 の 7 役割。画面ごとの「閲覧 / 更新 / 作成」を細かく設定可能
状態の進み方ルール
ステータスの遷移ルール。「仮予約 → 確定」「対応不可 → キャンセル」など、許可遷移を画面で定義

6.FAQ

よくいただく質問への回答です。

Q. 既存の業務システムや Excel との違いは何ですか？

段取りくんは 店間移動 + 業者連携を含む整備業務に特化した設計です。一般的な業務システムは整備伝票管理までは対応しますが、店間ピット共有や業者の自動通知・進捗追跡を 1 つにまとめている点が大きな違いです。

Excel では実現できない領域として、①リアルタイムの他店舗ピット空き状況参照、②業者への自動通知と進捗追跡、③同時編集の競合検出、④全操作の改ざん不可な記録、があります。

Q. 業者がスマホやパソコンを使えない場合はどうなりますか？

業者ポータルはスマホで動作するよう設計されていますが、業者の高齢化等で操作が難しい場合は以下の運用が可能です：

  • 店舗スタッフが業者と電話でやり取りし、店舗側で 業者の代理進捗更新を行う
  • 業者ごとに「通知方法」を email のみ、マイページのみ、両方から選択可能
  • 回送依頼書 PDF を印刷して業者へ FAX 送付する運用も併用可能

業者ポータルの利用度合いに応じて段階的に移行できます。

Q. 顧客（特に高齢者）が Web 予約を使えない場合は？

顧客は Web 予約だけが手段ではありません。電話受付の顧客に対しては、店舗スタッフが 「電話受付モード」で代行入力します。代行入力された予約も同じデータベースで一元管理されるため、Web 経由でも電話経由でも同じ画面で管理できます。

S01 シナリオを参照してください。

Q. 多店舗（10 店舗以上）でも使えますか？

はい。多店舗縮小ビュー（C.2+）で 12 店舗以上を 1 画面表示できます。各店舗のフィルタ・縦並び表示・要対応バッジで本部視点の管理が破綻しません。

権限は店舗ごと・役割ごとに細かく設定でき、店長は自店舗のみ、本部管理者は全店舗、という分離も可能です。

Q. データはどこに保管されますか？セキュリティは？

貴社専用のクラウド領域でデータを保管します。外部からのアクセス遮断、すべての操作を改ざん不可で記録する設計です。

個人情報は暗号化して保管し、保管期間も画面から設定できます。個人情報保護法・特定商取引法に基づく運用に対応します。詳細は §3.4 セキュリティとデータ保護を参照。

Q. 業者が指示に従わなかった場合の責任はどうなりますか？

業者の「対応可」回答時に 送信時刻・IP アドレス・同意チェックを自動記録し、操作記録に残します。回送依頼書 PDF にも責任分界文言を印字するため、業務委託契約の証跡となります。

業者の進捗が引取予定 + 3 時間経過しても更新されない場合、店舗側にアラートが飛び、本部管理者にエスカレーションされます（S05 参照）。

Q. 通知（メール）が業者に届かない場合はどうなりますか？

段取りくんは 送信失敗を自動検出し、最大 5 回まで自動再送します。それでも失敗した場合は「通知の再送・確認」画面（C.8）に失敗一覧として表示され、運用担当者が原因（メアド誤り・プロバイダ障害等）を確認して手動再送できます。

連続失敗が一定数を超えると本部管理者にエスカレーション通知が飛びます。詳細は S07 シナリオを参照。

Q. 導入にはどのくらいかかりますか？

機能スコープにもよりますが、コア機能（ピット共有・店間整備・業者通知・顧客予約）の MVP リリースは 14〜20 週間が目安です。マスター設定の整備（業者・レーン・店舗情報）は導入準備期間に並行で進められます。

詳細は別途お見積もり時にご相談ください。

Q. 既存の整備伝票データや顧客データを移行できますか？

CSV エクスポート / インポート機能で既存データの取り込みが可能です。移行対象となる典型データ：顧客 / 車両 / 整備伝票 / 業者マスター / 作業メニュー。

既存システムによっては API 連携や個別変換ツールが必要な場合もあります。詳細は導入支援時に個別対応いたします。

Q. 障害が起きた場合のサポートは？

本資料の範囲では具体的なサポート SLA は確定していません。導入時に以下を別途お決めいたします：

  • 一次対応の連絡先（メール / 電話）
  • 対応時間帯（営業時間内 / 24 時間）
  • 障害発生時の代替運用フロー（電話受付・紙ベース継続等）
  • 復旧目標時間（RTO） / データ復旧目標時点（RPO）
