## ★視覚確認済み: `c2-calendar.png` (ピット予約カレンダー・縮小表示モード)
実際に画像を確認した結果、**ユーザー指摘の「店舗別ピット稼働状況」はこのデザインで既に解決されている**。現実装(全予約1タイムライン)とデザインの乖離が「簡素すぎる」の正体。

確認できたレイアウト:
- **ヘッダ**: 段取りくんロゴ / 会社セレクタ「○○モータース」/ 検索バー(車両・整備伝票・顧客) / 通知ベル / 設定 / ユーザー名。
- **左ナビ (= 意図された正しいIA。現 admin-shell と異なる・業務志向)**:
  ホーム / カレンダー / 顧客予約 / 店間整備依頼(バッジ) / 業者通知・回送(バッジ) / 整備伝票 /
  **今日の工場ボード** / 一覧表 / (区切り) / 通知の再送・確認(バッジ) / 操作記録 / 設定。
  → 「今日の工場ボード」が独立ナビ項目として存在 (= `c6-floor` のフロアボード)。
  → 現状のマスター羅列ナビ(Phase 67で是正したもの)とは設計思想が違う。設定系はすべて「設定」配下に隠す IA。
- **本体**: タイトル「ピット予約カレンダー」「12店舗・縮小表示モード」/ 日・週・月切替 / 日付 / 詳細表示⇔縮小表示トグル。
- **店舗カード grid (2列, 1カード=1店舗)**: 各カードに
  - 店舗名 + レーン数 (例: 渋谷店 3レーン)
  - **稼働率バー + 「予約件数/容量・稼働率%」** (例: 18/24件・75%)
  - 稼働時間 (例: 8時間稼働)
  - 店間移動件数 (例: 店間2件)
  - **⚠要対応 バッジ** (例: 横浜店 要対応1)
  - 実例: 渋谷店 18/24・75% / 横浜整備工場 28/32・88% / 八王子店 6/16・38% / 相模原店 ⚠要対応1 21/24・88% など 12店舗。
- 「詳細表示」トグルで、おそらくレーン別タイムライン(§1.2 レーン別)に切替。

→ **= 店舗別ピット稼働ビューのデザインは確定済み**。実装すべきは「reservations + lanes + lane_working_hours 集計 → 店舗カード(稼働率/件数/店間/要対応)」。スキーマ充足は確認済(precheck)なので即実装可能。

### 意図されたナビIA (要・現実装との差分対応)
現 `admin-shell.tsx` はマスター/設定を全部トップに羅列。デザインは業務7項目 + 設定集約。**ナビIAの作り直し**もβ作業に含めるべき(現状はPhase67の暫定是正)。

## β計画への含意
1. **UIモック生成は完全に不要** → ワークフローから「デザイン工程」を削除できる。
2. **デザイン主導で実装可能** → 各機能の実装/委任時にPNGを渡す。仕様の曖昧さが減り、Codex委任の精度が上がる(独立性↑・仕様明確性↑ = 委任向き)。
3. ★ **核心の「店舗別ピット稼働」は `c6-floor` / `c1-dashboard` / `c2-calendar` にデザイン済み**。phase-67で「ゼロ実装」とされた中核に、明確なデザインターゲットがある。
4. 未実装画面(会社設定/監査ログ閲覧/顧客予約一覧/印刷PDF)も全てデザイン基準あり → 迷いなく実装可能。
